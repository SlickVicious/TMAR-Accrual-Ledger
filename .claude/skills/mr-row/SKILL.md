# Master Register Row Builder & Validator

The Master Register is **35 columns (A–AI) in strict order**. GAS writes/reads by
index, so a row array must always have exactly 35 entries in this order. Use `""`
for empty cells. The full schema and allowed values live in
`.claude/docs/domain-models.md` — load it if you need detail beyond this card.

## Column order (index → field)
1 ROW_ID · 2 DATE_ADDED · 3 PROVIDER · 4 MAILING_ADDRESS · 5 PROVIDER_EIN ·
6 ACCOUNT_NUMBER · 7 ACCOUNT_TYPE · 8 ACCOUNT_SUBTYPE · 9 ACCOUNT_AGENT ·
10 AGENT_ADDRESS · 11 STATUS · 12 OPENED_DATE · 13 CLOSED_DATE · 14 CURRENT_BALANCE ·
15 HIGH_BALANCE · 16 MONTHLY_PAYMENT · 17 APR_RATE · 18 BILLING_FREQUENCY ·
19 NEXT_PAYMENT_DUE · 20 PRIMARY_USER · 21 SECONDARY_USER · 22 ACCOUNT_PURPOSE ·
23 DOCUMENT_LOCATION · 24 LAST_VERIFIED · 25 LINKED_MR_ACCOUNT · 26 TRUST_ASSIGNMENT ·
27 TAX_RELEVANCE · 28 TAX_FORM · 29 DEDUCTION_TYPE · 30 CREDIT_REPORT_STATUS ·
31 REMOVAL_DATE · 32 DISPUTE_STATUS · 33 NOTES · 34 SOURCE · 35 DISCOVERY_STATUS

## Allowed enum values (validate these)
- **ACCOUNT_TYPE (7)**: Bank Account - Checking, Bank Account - Savings, Credit Card,
  Loan, Trust Account, Investment, Other
- **STATUS (6)**: Active, Closed, Pending, Disputed, Verified, Unverified
- **BILLING_FREQUENCY**: Monthly, Quarterly, Annual, Semi-Annual, Bi-Weekly, Weekly, On-Demand
- **PRIMARY_USER / SECONDARY_USER**: Clint, Syrina, Joint
- **TAX_FORM**: Form 1099-B, Schedule A, Form 990, Form 1065, Form 1120, Form 1041,
  Form 1120-S, Form 1120-W, Schedule C
- **DISCOVERY_STATUS**: Known, Unknown, Suspected, Verified
- **DISPUTE_STATUS** progression: "" → Disputed → Removed | Resolved
  (REMOVAL_DATE must be set when DISPUTE_STATUS = Removed)

## Rules
- **ROW_ID** format `MR-XXX` (zero-padded 3 digits). Never reuse or hand-pick — it is
  generated by incrementing the last ROW_ID. When generating, ask for the current last
  ROW_ID or read it; do not invent one.
- **CURRENT_BALANCE / HIGH_BALANCE / MONTHLY_PAYMENT** are numbers. Balance > 0 = asset,
  < 0 = liability.
- Dates are ISO strings.
- A new enum value is NOT allowed until the `_Validation` sheet dropdown is updated via
  `PopulateValidation.gs`.

## How to use
**Validate**: given an account object or proposed row, check (a) exactly 35 values in
order, (b) every enum field is within its allowed set, (c) ROW_ID matches `MR-\d{3}`,
(d) numeric fields are numbers, (e) REMOVAL_DATE present iff DISPUTE_STATUS = Removed.
Report each violation as `column N (FIELD): problem`.

**Build**: map a partial account object onto the 35-column array, filling unknowns with
`""`, normalizing enums to the canonical casing above, and leaving ROW_ID to the
generator unless the caller supplies the next ID. Output a ready-to-write JS array of 35
values plus a short note of any field you normalized or left blank.

Never reorder columns. Never output fewer or more than 35 values.
